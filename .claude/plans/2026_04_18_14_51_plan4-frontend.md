# Plan 4 — Frontend

## A. Context
The assessment requires a React 18 + TypeScript + Vite frontend with 4 pages, Zustand for auth state, React Query for server state, and specific UX features (status badges, conditional action buttons, stats display, loading/error states). JWT must be stored in memory (Zustand) — not localStorage — per assessment. This plan covers the complete frontend implementation.

## B. File-Level Changes

**File:** `packages/client/src/main.tsx` — CREATE
- React root render, wrap with `QueryClientProvider`, `BrowserRouter`

**File:** `packages/client/src/App.tsx` — CREATE
- Routes: `/login`, `/campaigns`, `/campaigns/new`, `/campaigns/:id`
- `PrivateRoute` wrapper: redirects to `/login` if no token in Zustand store

**File:** `packages/client/src/store/authStore.ts` — CREATE
- Zustand store: `{ token, user, setAuth, clearAuth }`
- `setAuth(token, user)` stores in memory only (no localStorage)

**File:** `packages/client/src/api/client.ts` — CREATE
- Axios instance with baseURL from env
- Request interceptor: attach `Authorization: Bearer <token>` from Zustand store
- Response interceptor: on 401, call `clearAuth()` and redirect to `/login`

**File:** `packages/client/src/api/campaigns.ts` — CREATE
- Typed functions: `listCampaigns(page)`, `getCampaign(id)`, `createCampaign(data)`, `updateCampaign(id, data)`, `deleteCampaign(id)`, `scheduleCampaign(id, scheduledAt)`, `sendCampaign(id)`, `getCampaignStats(id)`

**File:** `packages/client/src/api/recipients.ts` — CREATE

**File:** `packages/client/src/api/auth.ts` — CREATE

**File:** `packages/client/src/pages/LoginPage.tsx` — CREATE

## D. Verification Steps

2. Open `http://localhost` → redirected to `/login`
3. Login with seed credentials (`demo@example.com` / `password`) → redirect to `/campaigns`
4. List page shows 3 campaigns with correct status badges
5. Click draft campaign → detail page shows action buttons (Schedule, Delete)
6. Schedule campaign → badge changes to blue
7. Send campaign → badge transitions to yellow then green; stats bars update
8. Create new campaign via `/campaigns/new` → appears in list
9. Logout (clear token) → redirect to `/login`
